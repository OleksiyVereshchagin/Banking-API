# Banking API

RESTful API для банківської системи, розроблений на базі **Java** та **Spring Boot**.
Проект реалізує ключові банківські операції з урахуванням технічних викликів фінансових систем: захист від double-spending, ідемпотентність та правильну роботу з транзакціями.

## Основні можливості

- **Управління користувачами та безпека**:
    - Автентифікація на основі JWT (Access та Refresh токени).
    - Рольова модель доступу (USER, ADMIN).
    - Хешування паролів за допомогою BCrypt.
    - Оновлення профілів та керування паролями користувачів.
- **Операції з рахунками**:
    - Генерація IBAN згідно з українськими національними стандартами (29 символів, префікс UA).
    - Підтримка різних типів рахунків та керування їхнім статусом (Активний, Заблокований, Закритий).
- **Керування картками**:
    - Генерація номерів карток за алгоритмом Луна (Luhn).
    - Встановлення та зміна PIN-коду з захистом від перебору (Rate-limiting).
    - Маскування номерів карток для безпечного відображення даних.
- **Транзакції та перекази**:
    - Грошові перекази між рахунками, депозити та зняття коштів.
    - Підтримка ідемпотентності для запобігання дублюванню транзакцій.
    - Історія транзакцій з розширеною фільтрацією та пагінацією.
- **Документація та тестування**:
    - Повна специфікація API через OpenAPI 3.0 (Swagger UI).
    - Покриття коду тестами (JUnit 5, Mockito) та навантажувальне тестування за допомогою k6.

## Технологічний стек

- **Мова**: Java 
- **Фреймворк**: Spring Boot 
- **Безпека**: Spring Security, JWT 
- **Доступ до даних**: Spring Data JPA, Hibernate
- **База даних**: PostgreSQL
- **Документація**: OpenAPI 
- **Валідація**: Jakarta Bean Validation
- **Тестування**: JUnit 5, Mockito, **k6**
- **Збірка**: Maven
